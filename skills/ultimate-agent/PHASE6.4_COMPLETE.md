## 🎉 新增功能详情

### 1. 密码强度验证 🔐

**验证规则**:
- ✅ 最小长度：8 字符
- ✅ 最大长度：128 字符
- ✅ 字符类型要求（至少 3 种）:
  - 大写字母 (A-Z)
  - 小写字母 (a-z)
  - 数字 (0-9)
  - 特殊字符 (!@#$%^&*...)
- ✅ 常见弱密码检测（password/admin/123456 等）

**API 端点**:
```
POST /api/password-strength
Body: { "password": "..." }
Response: { "valid": true/false, "message": "..." }
```

**前端集成**:
- 实时密码强度检查
- 强度条可视化（弱/一般/好/强）
- 创建用户/修改密码时强制验证

**强度等级**:
```
弱    (红色) - < 6 字符
一般  (橙色) - 6-8 字符
好    (蓝色) - 8+ 字符，2 种类型
强    (绿色) - 8+ 字符，3+ 类型 ✅
```

---

### 2. 登录失败限制（防爆破）💥

**限制策略**:
- ✅ 最大失败尝试：5 次
- ✅ 时间窗口：15 分钟
- ✅ 锁定时长：30 分钟
- ✅ 自动解锁

**工作流程**:
```
1. 用户登录失败 → 记录时间戳
2. 15 分钟内失败 ≥ 5 次 → 账户锁定
3. 锁定 30 分钟 → 自动解锁
4. 登录成功 → 清除失败记录
```

**用户提示**:
```
登录失败时显示：
"用户名或密码错误 (剩余尝试：X 次)"

账户锁定时显示：
"账户已锁定，请 X 分钟后再试"
```

**安全特性**:
- 失败尝试内存存储（重启清除）
- 精确到秒的时间戳
- 动态计算剩余尝试次数
- 审计日志记录所有失败尝试

### 3. Token 自动续期 🔄

**续期机制**:
- ✅ Token 有效期：24 小时
- ✅ 续期方式：API 调用
- ✅ 旧 Token 撤销
- ✅ 新 Token 生成

**API 端点**:
```
POST /api/auth/refresh
Headers: Authorization: Bearer <token>
Response: { "token": "...", "expires_at": "..." }
```

**前端集成**:
- 账户设置页面手动刷新
- 自动续期（建议实现）:
  ```javascript
  // 每 23 小时自动续期
  setInterval(refreshToken, 23 * 60 * 60 * 1000);
  ```

**使用场景**:
- 用户长时间在线
- Token 即将过期
- 切换设备/浏览器

## 🚀 使用说明

### 修改密码
```
1. 访问 http://127.0.0.1:5000/settings.html
2. 输入原密码
3. 输入新密码（强度必须为"强"）
4. 确认新密码
5. 点击"保存修改"
6. 自动跳转登录页面
7. 使用新密码登录
```

### 刷新 Token
```
1. 访问账户设置页面
2. 点击"🔄 刷新 Token"
3. Token 有效期延长 24 小时
```

### 密码强度要求
```
✅ 强密码示例:
- MyP@ssw0rd123
- Secure#2026!
- OpenCl@w888

❌ 弱密码示例:
- password
- admin123
- 12345678
- qwerty
```
