## 🔄 标准工作流

### 1. 新产品开发流程
```
产品经理 → 架构师 → UI 设计师 → 高级开发 → 测试工程师 → 运维工程师
   ↓          ↓          ↓          ↓          ↓          ↓
  PRD      技术方案    界面设计    代码实现    测试验证    部署上线
```

### 2. Bug 修复流程
```
测试工程师 → 高级开发 → 测试工程师 → 运维工程师
    ↓          ↓          ↓          ↓
  Bug 定位    代码修复    回归测试    热修复部署
```

### 3. 数据分析流程
```
数据分析师 → 数据分析师 → 数据分析师 → 产品经理
    ↓          ↓          ↓          ↓
  数据收集    统计分析    可视化报表   业务洞察
```

### 4. 招聘流程
```
人力资源 → 人力资源 → 高级开发 → 人力资源
   ↓         ↓         ↓         ↓
 简历筛选   面试安排   技术面试   Offer 发放
```

## 💼 使用场景

### 场景 1: 新产品开发
**用户输入**: "我们要开发一个电商小程序"

**系统响应**:
1. CEO 协调器接收任务
2. 自动选择"新产品开发流程"
3. 依次调用:
   - 产品经理：撰写 PRD 和用户需求
   - 架构师：设计技术架构和 API
   - UI 设计师：制作界面原型
   - 高级开发：实现核心功能
   - 测试工程师：编写测试用例
   - 运维工程师：配置部署环境

### 场景 2: 数据分析请求
**用户输入**: "分析上季度的销售数据"

**系统响应**:
1. CEO 协调器分配给数据分析师
2. 数据分析师:
   - 收集和清洗数据
   - 进行统计分析
   - 生成可视化报表
   - 输出业务洞察

### 场景 3: 紧急 Bug 修复
**用户输入**: "线上支付功能出问题了"

**系统响应**:
1. CEO 协调器启动"Bug 修复流程"
2. 测试工程师：复现和定位问题
3. 高级开发：紧急修复代码
4. 测试工程师：回归测试
5. 运维工程师：热修复部署

## 🔧 配置说明

### 配置文件位置
- **主配置**: `configs/company-agents.json`
- **代理工作空间**: `workspaces/{agent_id}/`
- **会话状态**: `workspaces/{agent_id}/SESSION-STATE.md`

### 添加新代理
1. 在 `configs/company-agents.json` 的 `agents` 数组中添加
2. 创建对应的工作空间目录
3. 初始化 `SESSION-STATE.md`

### 自定义工作流
在 `configs/company-agents.json` 的 `workflows` 数组中定义:
```json
{
  "name": "自定义流程",
  "id": "workflow-custom",
  "steps": [
    {"agent": "agent-pm", "task": "需求分析"},
    {"agent": "agent-senior-dev", "task": "开发实现"}
  ]
}
```
