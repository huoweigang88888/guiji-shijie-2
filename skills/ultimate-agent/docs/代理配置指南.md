# 代理配置指南

**适用版本**: 公司多智能体系统 v1.0.0  
**配置文件**: `configs/company-agents.json`

---

## 📁 配置文件结构

```json
{
  "name": "公司场景多智能体系统",
  "version": "1.0.0",
  "orchestrator": { ... },  // CEO 协调器配置
  "agents": [ ... ],        // 代理列表
  "workflows": [ ... ],     // 工作流定义
  "communication": { ... }, // 通信配置
  "memory": { ... }         // 记忆配置
}
```

---

## 🔧 配置代理

### 添加新代理

在 `agents` 数组中添加：

```json
{
  "id": "agent-legal",
  "name": "法务顾问代理",
  "role": "Legal Advisor",
  "description": "合同审查、法律咨询、风险评估",
  "capabilities": [
    "合同审查",
    "法律咨询",
    "风险评估",
    "合规检查"
  ],
  "model": "bailian/qwen3.5-plus",
  "workspace": "workspaces/legal",
  "triggers": ["合同审查", "法律咨询", "合规问题"]
}
```

**必填字段**：
| 字段 | 说明 | 示例 |
|------|------|------|
| `id` | 代理唯一标识 | `agent-legal` |
| `name` | 显示名称 | `法务顾问代理` |
| `role` | 角色英文名称 | `Legal Advisor` |
| `description` | 职责描述 | `合同审查、法律咨询` |
| `capabilities` | 能力列表 | `["合同审查", ...]` |
| `model` | 使用的模型 | `bailian/qwen3.5-plus` |
| `workspace` | 工作空间目录 | `workspaces/legal` |

**可选字段**：
| 字段 | 说明 | 示例 |
|------|------|------|
| `triggers` | 触发关键词 | `["合同审查", ...]` |
| `priority` | 优先级 | `high/normal/low` |

---

### 修改现有代理

找到对应代理的 `id`，修改其配置：

```json
{
  "id": "agent-pm",
  "name": "高级产品经理代理",  // 修改名称
  "capabilities": [            // 添加能力
    "需求收集",
    "竞品分析",
    "PRD 撰写",
    "用户故事",
    "优先级排序",
    "数据分析"                 // 新增
  ]
}
```

---

### 删除代理

从 `agents` 数组中移除对应的 JSON 对象即可。

## 🔄 配置工作流

### 添加新工作流

在 `workflows` 数组中添加：

```json
{
  "name": "合同审批流程",
  "id": "workflow-contract-approval",
  "steps": [
    {"agent": "agent-pm", "task": "需求说明"},
    {"agent": "agent-legal", "task": "法律审查"},
    {"agent": "agent-finance", "task": "财务审核"},
    {"agent": "agent-ceo", "task": "最终批准"}
  ]
}
```

**工作流步骤**：
| 字段 | 说明 |
|------|------|
| `agent` | 代理 ID（如 `agent-pm`） |
| `task` | 该步骤的任务描述 |

---

### 修改工作流

```json
{
  "name": "新产品开发流程（简化版）",
  "id": "workflow-new-product-lite",
  "steps": [
    {"agent": "agent-pm", "task": "PRD"},
    {"agent": "agent-senior-dev", "task": "开发"},
    {"agent": "agent-qa", "task": "测试"}
  ]
}
```

## 📂 创建工作空间

每添加一个新代理，需要创建对应的工作空间：

```bash
# PowerShell
New-Item -ItemType Directory -Force -Path "skills/ultimate-agent/workspaces/legal"

# 创建 SESSION-STATE.md
@"
# Legal Advisor Agent - Session State

**Agent ID**: agent-legal  
**Role**: Legal Advisor  
**Status**: 🟢 Active  

---

## Current Tasks
- [ ] 等待任务分配

## WAL Log
*待记录*

## Memory
*待记录*
"@ | Out-File -FilePath "skills/ultimate-agent/workspaces/legal/SESSION-STATE.md" -Encoding UTF8
```

## ⚙️ 高级配置

### 1. 通信配置

```json
"communication": {
  "channels": ["webchat", "discord", "telegram"],
  "broadcast_on_completion": true,
  "daily_standup": "09:00",
  "weekly_report": "Friday 18:00"
}
```

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `channels` | 通信渠道 | `["webchat"]` |
| `broadcast_on_completion` | 任务完成时广播 | `true` |
| `daily_standup` | 每日站会时间 | `"09:00"` |
| `weekly_report` | 周报时间 | `"Friday 18:00"` |

---

### 2. 记忆配置

```json
"memory": {
  "shared_knowledge_base": "workspaces/shared/knowledge",
  "project_memories": "workspaces/{agent_id}/memory",
  "retention_days": 90
}
```

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `shared_knowledge_base` | 共享知识库路径 | `workspaces/shared/knowledge` |
| `project_memories` | 项目记忆路径模板 | `workspaces/{agent_id}/memory` |
| `retention_days` | 记忆保留天数 | `90` |

---

### 3. 模型配置

每个代理可以使用不同的模型：

```json
{
  "id": "agent-senior-dev",
  "model": "bailian/qwen3.5-plus"  // 高性能模型
}
```

**可用模型别名**：
| 别名 | 完整模型 | 适用场景 |
|------|----------|----------|
| `qwen` | `bailian/qwen3.5-plus` | 通用任务 |
| `codex` | `openai/codex` | 编程任务 |
| `claude-code` | `anthropic/claude-code` | 复杂编程 |

## 🚨 常见问题

### Q1: 添加代理后 Web UI 不显示？
**A**: 重启 Web UI 服务：
```bash
# 停止旧进程
Stop-Process -Name python -Force

# 重新启动
cd skills/ultimate-agent/web_ui
python -c "from app import run_dashboard; run_dashboard(port=18789)"
```

### Q2: 工作流不生效？
**A**: 检查 `workflow_id` 是否与任务类型匹配，或 CEO 协调器是否正确加载配置。

### Q3: 如何禁用某个代理？
**A**: 添加 `"enabled": false` 字段，或直接从配置中移除。
