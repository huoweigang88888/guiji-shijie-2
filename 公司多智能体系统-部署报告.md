## 🔄 标准工作流

### 1. 新产品开发流程
```
产品经理 → 架构师 → UI 设计师 → 高级开发 → 测试工程师 → 运维工程师
   ↓          ↓          ↓          ↓          ↓          ↓
  PRD      技术方案    界面设计    代码实现    测试验证    部署上线
```

### 2. Bug 修复流程
```
测试工程师 → 高级开发 → 测试工程师 → 运维工程师
    ↓          ↓          ↓          ↓
  Bug 定位    代码修复    回归测试    热修复部署
```

### 3. 数据分析流程
```
数据分析师 → 数据分析师 → 数据分析师 → 产品经理
    ↓          ↓          ↓          ↓
  数据收集    统计分析    可视化报表   业务洞察
```

### 4. 招聘流程
```
人力资源 → 人力资源 → 高级开发 → 人力资源
   ↓         ↓         ↓         ↓
 简历筛选   面试安排   技术面试   Offer 发放
```

## 💡 使用示例

### 示例 1: 新产品开发
**用户输入**: 
```
我们要开发一个电商小程序，包含商品展示、购物车、支付功能
```

**系统自动执行**:
1. CEO 协调器接收任务
2. 选择"新产品开发流程"
3. 依次调用 6 个代理完成各自环节
4. 最终交付完整产品

### 示例 2: 数据分析
**用户输入**: 
```
分析上季度的销售数据，生成可视化报表
```

**系统响应**:
- 分配给数据分析师代理
- 执行数据收集 → 分析 → 可视化 → 洞察
- 输出完整分析报告

### 示例 3: 紧急 Bug
**用户输入**: 
```
线上支付功能出问题了，用户无法完成付款
```

**系统响应**:
- 启动"Bug 修复流程"
- 测试工程师定位问题
- 高级开发紧急修复
- 运维工程师热修复部署
